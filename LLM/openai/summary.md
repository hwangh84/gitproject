# OpenAI 폴더 파일 요약

이 문서는 `openai` 폴더 내의 파일들에 대한 상세 요약입니다. 이 폴더는 LLM(Large Language Model)을 활용한 다양한 **프롬프트 엔지니어링 기법**과 이를 실제 **비즈니스 로직(고객센터)**에 적용하는 예시 코드를 담고 있습니다.

---

## 1. `01.ipynb`: 프롬프트 엔지니어링 기법 (Prompt Engineering Techniques)

이 파일은 LLM의 성능을 극대화하기 위한 다양한 프롬프트 기법들을 단계별로 실습하는 튜토리얼입니다.

### 기본 기법
*   **Zero-shot Prompting:** 예시 없이 지시사항만으로 작업을 수행하게 하는 가장 기본적인 방식. (예: 문장의 감정 분류)
*   **Few-shot Prompting:** 모델에게 몇 가지 예시(Shot)를 제공하여 패턴을 학습시키고 성능을 높이는 방식. (예: 단어를 이모지로 변환)
*   **Chain-of-Thought (CoT):** "생각의 사슬"이라는 뜻으로, 단순히 답만 내는 것이 아니라 문제 해결 과정을 단계별로 서술하게 하여 복잡한 추론 능력을 향상시키는 기법. (예: 수학 문제 풀이 과정 서술)

### 심화 기법
*   **Self-Consistency (자기 일관성):** 같은 질문을 여러 번 던져서 가장 많이 나온 답변을 채택하여 신뢰도를 높이는 방식.
*   **Generated Knowledge Prompting (지식 생성):** 질문에 바로 답하기 전에, 관련된 지식을 먼저 생성하게 하고 그 지식을 바탕으로 답변하게 하여 정확도를 높이는 기법. (예: 골프 홀인원이 어려운 이유 설명 전 골프 지식 생성)
*   **Prompt Chaining:** 복잡한 작업을 한 번에 처리하지 않고, 여러 단계로 나누어 앞 단계의 결과를 다음 단계의 입력으로 사용하는 파이프라인 방식. (예: 이메일에서 주제 추출 -> 답장 작성)

### 고급/최신 기법
*   **RAG (Retrieval Augmented Generation):** LLM이 모르는 외부 데이터(사내 문서 등)를 검색(Retrieval)하여 프롬프트에 참고 자료로 넣어주는 기술. (예: 회사 재택근무 규정 참조 답변)
*   **Automatic Reasoning and Tool-use:** LLM이 스스로 계산기나 날씨 API 등 외부 도구가 필요한지 판단하고 호출하는 에이전트(Agent)의 기초 개념.
*   **APE (Automatic Prompt Engineer):** 사람이 프롬프트를 짜는 것이 아니라, LLM에게 "이 작업을 위한 완벽한 프롬프트를 짜줘"라고 시키는 역발상 기법.
*   **Active-Prompt:** LLM이 답변하기 어려워하거나 확신도(Confidence)가 낮은 문제를 찾아내어, 사람에게 예시를 요청하거나 Few-shot 예제로 추가하여 성능을 점진적으로 개선하는 방식.
*   **Directional Stimulus Prompting:** 구체적인 지시 전에 힌트(키워드)를 주어 답변의 방향을 유도하는 기법. (예: 기사 요약 시 '소프트웨어', 'AI' 키워드 중심으로 요약 유도)
*   **PAL (Program-Aided Language Models):** 수학 문제나 날짜 계산 등 LLM이 약한 부분을 파이썬 코드를 생성하여 실행함으로써 해결하는 방식.
*   **ReAct (Reason + Act):** 생각(Reason), 행동(Act), 관찰(Observation)의 루프를 통해 문제를 해결하는 에이전트 설계 방식.
*   **Reflexion (반성):** 틀린 답변을 냈을 때, 왜 틀렸는지 스스로 분석(반성)하고 수정된 답변을 내놓게 하는 자가 수정 기법.
*   **Multimodal CoT:** 텍스트뿐만 아니라 이미지 정보를 함께 활용하여 단계별로 추론하는 기법.
*   **Graph Prompting:** 데이터를 텍스트가 아닌 그래프(노드와 관계) 형태로 표현하여 관계 추론 능력을 향상시키는 기법.
*   **Meta-prompting:** 복잡한 문제를 해결하기 위해 LLM이 스스로 하위 전문가(Sub-LLM)들을 정의하고, 각 전문가에게 업무를 배분한 뒤 결과를 종합하는 '지휘자' 역할을 수행하게 하는 기법.

## 2. `02.ipynb`: AI 기반 고객센터 자동화 (AI Customer Support Agent)

이 파일은 앞서 배운 기법들(특히 Prompt Chaining, RAG 개념)을 활용하여 **실제 고객센터 업무를 자동화**하는 시나리오를 구현했습니다.

### 주요 프로세스
1.  **고객 리뷰 분석 (Analysis):**
    *   고객의 리뷰 텍스트를 입력받아 **감정(긍정/부정)**, **카테고리(배송/품질 등)**, **요구사항(환불/교환 등)**, **핵심 불만**을 JSON 형태로 구조화하여 추출합니다.
2.  **의사결정 (Decision Making - RAG & CoT):**
    *   **회사 규정(Policy Document)**을 지식 베이스로 제공합니다. (단순 변심 7일, 불량 30일, AS 1년 등)
    *   LLM에게 "CS 매니저" 페르소나를 부여하고, 고객 데이터(구매일, 박스 유무 등)와 회사 규정을 비교하여 **승인/거절 여부**를 판단하게 합니다.
    *   단순 판단이 아니라 **CoT(단계별 추론)**를 통해 규정 몇 조에 해당하는지, 날짜 계산은 맞는지 논리적으로 따져보게 합니다.
3.  **이메일 초안 작성 (Drafting):**
    *   판단 결과(승인/거절)에 따라 톤앤매너를 조절합니다.
    *   **승인 시:** 친절하고 빠른 안내.
    *   **거절 시:** 안타까워하며 공감하되 원칙을 단호하게 설명하고, 대안(AS, 적립금 등)을 제시하는 정중한 이메일을 자동 생성합니다.

### 요약
*   **`01.ipynb`**: **"도구 상자"**로서 다양한 LLM 활용 기술을 하나씩 실습해보는 파일.
*   **`02.ipynb`**: 그 도구들을 조합하여 **"고객센터 자동화"**라는 구체적인 비즈니스 문제를 해결하는 **실전 응용 프로젝트**. 단순 챗봇을 넘어 규정을 해석하고 판단까지 내리는 **에이전트(Agent)**의 모습을 보여줍니다.
